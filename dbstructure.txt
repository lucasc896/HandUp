### DB Structure ideas ###
- User
    - id
    - Name
    - Unique ID
    - Date of Birth
    - Date added
    - Place of Birth
- User History
    - Foreign key to User.id
    - Last time seen
    - Last seen area
    - Last institution visited
    - Last institution date
- User Balance
    - Foreign key to User.id
    - Transaction Date
    - Transaction Type
    - Transaction Location (null if a donation?)
    - Transaction Amount
- Institution
    - id
    - Name
    - Address
    - Contact number
    - Email address
    - Type
- Institution Payment Details
    - Foreign key 
    - Account Number
    - Sort Code
    - Account Type
